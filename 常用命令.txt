25. 在windows系统上对一个c++可执行文件弄成开机自启
通过注册表设置：
1.按下Win + R组合键，输入regedit，回车打开注册表编辑器。
2.如果是当前用户自启，定位到HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run；如果是所有用户自启，定位到HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run。
3.右侧空白处右键，选择 “新建”>“字符串值”，命名为合适的名称，比如ServerExe。
4.双击新建的字符串值，在 “数值数据” 栏中输入server.exe的完整路径，例如C:\Program Files\Server\server.exe。

[Unit]
Description=Restart Video Analysis at 06:00 daily
Requires=video-analysis.service

[Timer]
OnCalendar=*-*-* 06:00:00
Persistent=true
Unit=video-analysis-restart.service

[Install]
WantedBy=timers.target
EOF

sudo tee /etc/systemd/system/video-analysis-restart.service > /dev/null << 'EOF'
[Unit]
Description=Restart Video Analysis Service
After=video-analysis.service

[Service]

User=orin
Group=orin
ExecStart=/usr/bin/systemctl restart video-analysis.service
StandardOutput=journal
StandardError=journal

31.查看内存消耗情况
watch -n 5 "ps -p 10168 -o rss,vsz,pcpu,cmd"

# 每隔 5 秒显示 PID 10168 的内存（MB）、CPU、命令
watch -n 5 "ps -p 9447 -o rss,vsz,pcpu,cmd | awk 'NR==1{print \"RSS(MB)\\tVSZ(MB)\\tCPU(%)\\tCMD\"} NR>1{printf \"%.2f\\t%.2f\\t%s\\t%s\\n\", \$1/1024, \$2/1024, \$3, \$4}'"
终止：

1） RSS (Resident Set Size) - 常驻内存集
当前值: 2,625,068 KB ≈ 2.5 GB
含义: 进程实际使用的物理内存大小
说明: 这部分是真正加载到RAM中的内存，不包括交换分区(swap)

2） VSZ (Virtual Memory Size) - 虚拟内存大小
当前值: 14,688,408 KB ≈ 14 GB
含义: 进程可以访问的总虚拟地址空间
说明: 包括物理内存、交换文件、共享库等所有映射的内存区域

3） %CPU - CPU 使用率
当前值: 134%
含义: 进程占用CPU核心的百分比
说明:
	100% = 1个核心满载
	134% = 约1.34个核心被持续使用
	这是多线程程序的典型特征

4） CMD - 命令名称
值: ./video_analysis
含义: 正在运行的进程名称

# 验证 RTC 硬件时钟是否写入成功
sudo hwclock -r


33.系统时间同步：
win10服务器：
	1.使用win + R 组合键在运行窗口中输入regedit；
	2.找到目录：\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\W32Time\TimeProviders\NtpServer，把Enabled设置为1 ；
	3.计算机\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\W32Time\Config，并把AnnounceFlags的值设置为5（系统默认为a）；
	4.使用win + R 组合键在运行窗口中输入services.msc，打开服务；
	5.在服务项中找到Windows Time ；
	6.设置为自动后确定，点击右键重新启动此服务，至此服务启动成功；
	7.启动和关闭windows NTP的方式：net stop w32time；net start w32time；
	8.防火墙放行 NTP 端口：netsh advfirewall firewall add rule name="NTP Server" dir=in action=allow protocol=UDP localport=123
	8.本机测试：在cmd窗口中输入w32tm /stripchart /computer:127.0.0.1 ，如果有回显则服务正常。
jeston-ubuntu客户端：
在线安装：
	1.安装：sudo apt-get install chrony；
	2.修改配置文件：vim /etc/chrony.conf；
	 添加以下内容：
		# /etc/chrony/chrony.conf

		# 指定时间源（Win10工作站）
		server 172.20.31.57 iburst minpoll 5 maxpoll 8
		# minpoll 5 = 32秒最小间隔
		# maxpoll 8 = 256秒最大间隔
		# iburst = 初始快速同步几次

		# 记录系统时钟的漂移率
		driftfile /var/lib/chrony/drift

		# 允许时间在启动时快速调整
		makestep 1.0 3

		# 如果无法连接NTP服务器，仍保持时间
		local stratum 10

		# 日志
		log measurements statistics tracking
	3.重启服务：
		systemctl restart chronyd；
