## rtsp协议转webrtc协议播放视频流

### 一、ZLMediaKit 简介

```
	ZLMediaKit 是一款开源、高性能、跨平台的流媒体服务框架，由国内开发者基于 C++11 开发，核心目标是解决音视频流媒体的 “收、转、发” 问题 —— 简单说，它能接收各种协议的音视频流（如 RTSP/RTMP），转换成其他协议（如 WebRTC/HTTP-FLV/HLS），并高效分发给终端（浏览器 / 播放器）
	它不是单纯的 “播放器” 或 “推流工具”，而是流媒体服务器，类似 SRS、EasyDarwin，但在性能、易用性、跨平台性上有独特优势，尤其适合物联网、安防监控、直播等场景（这也是你用它做 RTSP 转 WebRTC 的核心原因）。
```

### 二、ZLMediaKit-RTSP 服务器

#### 1.1 克隆代码仓库

```
git clone https://gitcode.com/gh_mirrors/zlme/ZLMediaKit
cd ZLMediaKit
```

#### 1.2 初始化子模块

```
git submodule init
git submodule update --recursive
```

#### 1.3 安装依赖

```
sudo apt-get update
sudo apt-get install -y build-essential cmake
sudo apt-get install -y libssl-dev libsrtp2-dev
sudo apt-get install -y libavformat-dev libavcodec-dev libavutil-dev
```

#### 1.4 编译MediaServer

```
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j4
```

#### 1.5 修改配置文件

```
cd /home/supcon/alg/wdd/rtsp_to_webrtc/ZLMediaKit/release/linux/Release

```

```
  -H "Content-Type: application/json" \
  -d '{
    "secret": "123456",
    "vhost": "__defaultVhost__",
    "app": "live",
    "stream": "test_stream",
    "url": "rtsp://172.20.31.10:8555/stream",
    "rtp_type": 0,
    "enable_hls": 1,
    "enable_mp4": 0,
    "enable_rtsp": 1,
    "enable_rtmp": 1,
    "enable_fmp4": 1,
    "enable_ts": 0

```
